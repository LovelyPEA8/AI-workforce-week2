
# Problem Statement

## Overview
The IT Team at Fabrikam currently handles user support requests in an unstructured and inconsistent manner. Users often provide incomplete or unclear information when reporting issues, which leads to delays, repeated follow-ups, and inefficiencies in the support process.

## Problem Description
When employees encounter IT-related issues, they typically submit requests through informal channels such as emails, chats, or verbal communication. These methods lack a standardized format, making it difficult for the IT team to quickly understand and categorize the problem.


Key challenges include:
- Missing critical information (e.g., user identity, department, issue type)
- Inconsistent request formats across users
- Increased back-and-forth communication to gather required details
- Delay in resolving issues due to incomplete data
- Inefficient tracking and prioritization of IT requests

## Project Objective
This project aims to design and implement a conversational Copilot agent using Microsoft Copilot Studio to streamline the IT request submission process. The agent will:
- Collect essential user information such as name, department, and issue type
- Store the collected data using structured variables
- Present a clear and formatted summary of the request back to the user
- Ensure completeness and consistency before submission

## Expected Outcome
By implementing this solution, Fabrikam aims to:
- Improve data accuracy and completeness of IT requests
- Reduce response and resolution time
- Minimize back-and-forth communication

- Enhance user experience through guided interaction
- Establish a structured and scalable request intake process

## Scope
The solution will focus on:
- Designing a conversational flow for capturing user inputs
- Using variables to store and reuse information
- Displaying structured summaries within the Copilot interaction
- Testing the process within the Copilot Studio environment

## Conclusion
This capstone project addresses the inefficiencies in IT support request handling by introducing a structured, automated, and user-friendly conversational interface. The solution will help bridge the gap between users and the IT team, ensuring faster and more effective issue resolution.
